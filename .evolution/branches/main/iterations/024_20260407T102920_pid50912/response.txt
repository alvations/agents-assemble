All edits pass syntax validation. Summary of changes:

- **Fixed `fetch_multiple_ohlcv` batch path missing NaN Close filter** — Individual `fetch_ohlcv` drops rows with NaN Close, but the batch `yf.download` path only did `dropna(how="all")`, allowing rows with NaN Close into results. Added `dropna(subset=["Close"])` for consistency.
- **Fixed `fetch_fred_series` empty observations returning wrong structure** — Returned `pd.DataFrame(columns=["value"])` with a RangeIndex instead of a DatetimeIndex named "date", causing downstream code expecting `df.index` to be dates to break. Now returns a properly typed empty DataFrame.
- **Fixed `fetch_yield_curve` fetching years of unnecessary data** — When `date="2022-06-15"`, it fetched from 2021-06-15 to *today* (potentially years of data). Now passes `end=date+15days` to `fetch_fred_series`, limiting the fetch window.
- **Fixed `discover_universe_from_etf` silent failure** — The function was a stub that silently returned `[]`; now raises `NotImplementedError` so callers know it's unimplemented.
- **Fixed non-deterministic universe ordering** in `scan_52_week_lows` and `scan_volatile_stocks` — `list(set())` gives arbitrary iteration order; changed to `sorted()` for reproducible results.
